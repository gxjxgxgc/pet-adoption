## 2. 权限控制实现方案

### 2.1 技术方案

#### 2.1.1 后端权限控制

**实现方式**：
- **基于角色的访问控制（RBAC）**：使用Spring Security实现角色权限控制
- **注解方式**：使用`@PreAuthorize`注解在方法级别进行权限控制
- **过滤器**：使用JWT过滤器验证用户身份和权限
- **权限检查**：实现权限检查工具类，提供权限验证方法

**核心组件**：
- `JwtAuthenticationFilter`：JWT令牌验证和用户身份解析
- `SecurityConfig`：Spring Security配置
- `PermissionChecker`：权限检查工具类
- `@PreAuthorize`：方法级权限控制注解

#### 2.1.2 前端权限控制

**实现方式**：
- **路由守卫**：使用Vue Router的导航守卫进行路由权限控制
- **指令方式**：自定义指令实现按钮级权限控制
- **状态管理**：在Pinia store中管理用户权限信息
- **动态渲染**：根据用户权限动态渲染UI组件

**核心组件**：
- `router/index.js`：路由配置和导航守卫
- `directives/permission.js`：权限控制指令
- `stores/user.js`：用户状态和权限管理
- `components/Header.vue`：根据权限动态渲染导航

### 2.2 实现步骤

#### 2.2.1 后端实现

1. **配置Spring Security**
   - 配置`SecurityConfig`，设置认证和授权规则
   - 配置JWT过滤器，验证用户身份

2. **实现权限注解**
   - 使用`@PreAuthorize`注解标记需要权限的方法
   - 定义权限表达式，如`hasRole('ADMIN')`

3. **实现权限检查**
   - 创建`PermissionChecker`工具类
   - 实现权限验证方法

4. **数据库设计**
   - 设计用户角色表
   - 设计权限表
   - 设计角色权限关联表

#### 2.2.2 前端实现

1. **路由配置**
   - 在路由配置中添加`meta`字段，定义路由需要的权限
   - 实现导航守卫，检查用户权限

2. **权限指令**
   - 自定义`v-permission`指令
   - 根据用户权限控制元素的显示/隐藏

3. **状态管理**
   - 在`user` store中存储用户权限信息
   - 提供权限检查方法

4. **UI渲染**
   - 根据用户权限动态渲染导航菜单
   - 根据用户权限动态渲染操作按钮

## 3. 权限控制实现

### 3.1 后端实现

#### 3.1.1 SecurityConfig配置

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .csrf().disable()
            .sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            .and()
            .authorizeRequests()
            .antMatchers("/api/auth/**").permitAll()
            .antMatchers("/api/admin/**").hasRole("ADMIN")
            .antMatchers("/api/shelter/**").hasAnyRole("ADMIN", "SHELTER")
            .anyRequest().authenticated();
        
        http.addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
    }
}
```

#### 3.2.2 导航守卫

```javascript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore();
  
  if (to.meta.requiresAuth) {
    if (!userStore.isLoggedIn) {
      next('/login');
    } else if (to.meta.roles && !to.meta.roles.includes(userStore.userRole)) {
      next('/403');
    } else {
      next();
    }
  } else {
    next();
  }
});
```

#### 3.2.3 权限指令

```javascript
app.directive('permission', {
  mounted(el, binding) {
    const userStore = useUserStore();
    const requiredRoles = binding.value;
    
    if (requiredRoles && !requiredRoles.includes(userStore.userRole)) {
      el.style.display = 'none';
    }
  }
});
```

## 5. 权限管理最佳实践

### 5.1 安全建议

1. **最小权限原则**：只授予用户完成任务所需的最小权限
2. **权限分离**：不同角色的权限应该明确分离
3. **定期审查**：定期审查用户权限，确保权限分配合理
4. **密码策略**：实施强密码策略，定期更换密码
5. **审计日志**：记录权限相关操作，便于追溯和审计

### 5.2 性能优化

1. **缓存策略**：缓存用户权限信息，减少数据库查询
2. **批量检查**：批量检查权限，减少权限验证次数
3. **异步处理**：异步处理权限相关的非关键操作
4. **索引优化**：为权限相关的数据库查询添加索引

### 5.3 可扩展性考虑

1. **动态权限**：支持动态分配权限，无需修改代码
2. **权限模板**：预设权限模板，简化权限分配
3. **角色继承**：支持角色继承，减少权限配置复杂度
4. **权限分组**：支持权限分组，便于管理和维护

## 6. 总结

通过建立完整的角色权限体系和实现权限控制机制，宠物领养系统能够确保不同角色的用户只能访问和操作授权的功能和数据，提高系统的安全性和可管理性。

权限控制的实现遵循了以下原则：
1. **安全性**：确保只有授权用户能够访问和操作系统资源
2. **灵活性**：支持动态权限分配和角色管理
3. **可维护性**：权限配置清晰，易于管理和维护
4. **性能**：权限验证高效，不影响系统性能

这些措施将有助于保护系统数据安全，防止未授权访问，同时为不同角色的用户提供合适的功能访问权限，提升系统的整体用户体验。
